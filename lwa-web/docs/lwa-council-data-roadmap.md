## Clip Scoring Model

### 12-Component Weighted Formula

| Component | Weight | Raises Score | Lowers Score | Backend Field |
|---|---|---|---|---|
| Hook Score | 25% | Hook pattern match in first 15 words | Flat opener, no pattern match | `hook_score` |
| Retention Score | 20% | Even energy; strong final third | Front-loaded energy, dead zones | `retention_score` |
| Emotional Spike Score | 15% | Laughter, anger, surprise, inspiration | Monotone, no peak | `emotional_spike_score` |
| Clarity Score | 10% | Topic clear in <10s | Jargon, context-dependent | `clarity_score` |
| Platform Fit Score | 10% | Length/pacing matches platform | Wrong aspect, wrong pace | `platform_fit_score` |
| Visual Energy Score | 5% | Movement, expressions, visual elements | Static shot, poor lighting | `visual_energy_score` |
| Audio Energy Score | 5% | Clear audio, dynamic vocal range | Low volume, noisy background | `audio_energy_score` |
| Controversy Score | 3% | Contrarian claims, debate structure | Safe, universally agreed content | `controversy_score` |
| Educational Value Score | 3% | High fact density, instructional language | Vague motivation, no actionable insight | `educational_score` |
| Share/Comment Score | 2% | Question present, open loop, universal theme | Informative but not engaging | `share_comment_score` |
| Render Readiness Score | 1% | Good SNR, resolution, frame stability | Noise, low res, choppy audio | `render_readiness_score` |
| Commercial Value Score | 1% | Social proof, offer language, result quantified | Entertainment only, no commercial angle | `commercial_value_score` |

**Final formula:**
```
VIRALITY_SCORE = Σ (component_score × weight)
```

**Tier thresholds:**
- ≥ 75: Tier 1 (lead clip)
- 55–74: Tier 2 (strong clip)
- 35–54: Tier 3 (strategy-only)
- < 35: Archive

**Confidence score:**
```
CONFIDENCE = (transcript_quality × 0.4) + (audio_clarity × 0.3) + (signal_completeness × 0.3)
```

## Product Roadmap

### Phase 1 — MVP Intelligence Improvements (Now)

**Build:**
1. Hook pattern library (16 types) against transcript at clip generation
2. 12-component virality score with weighted formula
3. Render readiness pre-check before render queue
4. `strategy_only_reason` + `recovery_recommendation` fields

**Why:** These are the foundation. Without them, scoring is guesswork and render failures are invisible.

**Backend:** `hook_pattern_library.json`, `scoring_engine.py`, readiness check in render queue entry, schema migration for new fields.
**Frontend:** Display hook_score on clip card, score breakdown in advanced view, surface `strategy_only_reason` immediately post-upload.
**Test:** Hook detection 80%+ precision on 50 labeled clips. A/B: ranked clips vs. old ordering.

---

### Phase 2 — Better Ranking and Packaging

**Build:**
1. Post order ranking engine with diversity scoring
2. Hook + caption variant generation (3–5 per clip)
3. Platform-specific caption engine (7 platforms)

**Why:** Single captions, identical ordering, and no variants leave real performance on the table.

**Backend:** `diversity_engine.py`, `hook_variant_engine.py`, `caption_variant_engine.py`, `platform_caption_engine.py`.
**Frontend:** Post order badges, variants panel, platform selector auto-loads correct caption.
**Test:** 3 variants generated per clip. Variants are meaningfully different (cosine sim < 0.85). No 2 consecutive clips share speaker + topic.

---

### Phase 3 — Rendered Clip Quality and Captions

**Build:**
1. 5 caption style presets with auto-selection
2. Thumbnail auto-selection + text overlay
3. Export bundle: MP4 + SRT + PNG + caption.txt

**Why:** Caption style and thumbnail quality are the primary visual differentiators. Bad defaults hurt performance.

**Backend:** `caption_style_engine.py`, `thumbnail_engine.py`, `export_bundle.py`.
**Frontend:** Style badge on clip card, thumbnail preview with variant selector, Export All button.
**Test:** Auto-selected style correct 75%+ in user study. Thumbnail outperforms random frame on CTR.

---

### Phase 4 — Campaign / Agency Workflow

**Build:**
1. Multi-source batch processing with client management
2. White-label export and branding
3. Campaign-level post scheduling + basic analytics

**Why:** Agencies are the highest LTV segment. They need client isolation, white-label, and scheduling.

**Backend:** `client_id` in data model, batch upload API, branding config, campaign scheduler.
**Frontend:** Client selector, per-client history, brand settings panel, campaign calendar.
**Test:** 10 clients with 5 videos each — no data cross-contamination. White-label: zero LWA branding in export.

---

### Phase 5 — Trend and Always-Current Intelligence

**Build:**
1. Real-time trend signal integration (TikTok Creative Center, IG Trends)
2. Evergreen vs. time-sensitive classification

**Why:** Trend-aware clipping dramatically improves performance. Evergreen classification enables content calendar.

**Backend:** `trend_signal_service.py`, `reuse_potential` scoring, `time_sensitive` flag.
**Frontend:** Trend badge, evergreen badge, repost recommendations in calendar.
**Test:** Trend-matched clips outperform non-matched on 50-clip study.

---

### Phase 6 — Enterprise / Creator Team System

**Build:**
1. Multi-user team accounts with role-based access
2. Public REST API + webhooks
3. Creator analytics and performance feedback loop

**Why:** Enterprise accounts, developer integrations, and the feedback loop that makes scoring smarter over time.

**Backend:** `user_roles` table, API key management, rate limiting, webhook on render complete, analytics feedback engine.
**Frontend:** Team management, API docs, performance dashboard.
**Test:** Editor cannot publish without reviewer approval. API: full clip pipeline without UI.

## Council Executive Summary

### Top 10 Product Insights
1. Hook strength is the single most predictive signal. Score it first.
2. Strategy-only clips are assets, not failures. Package them fully.
3. Platform fit is not optional — same clip needs different treatment per platform.
4. Confidence score is what converts AI output into creator trust.
5. Caption style is a performance variable, not decoration.
6. Post order matters as much as clip quality.
7. "Best clip first" must be enforced in UI architecture — no scrolling to find it.
8. Whop and community content is underserved and high-value.
9. Evergreen classification is a retention and scheduling feature.
10. The competitor gap is at the intelligence layer. OpusClip detects. LWA explains.

### Top 10 Backend Requirements
1. `hook_score` from hook pattern library match on first 15 words
2. 12-component `virality_score` weighted composite
3. `render_readiness_score` pre-check with SNR, resolution, frame stability thresholds
4. `strategy_only_reason` and `recovery_recommendation` on all failed clips
5. `platform_fit` JSON per clip per platform
6. `recommended_post_order` from diversity-aware ranking engine
7. `hook_variants[]` and `caption_variants[]` (3–5 each)
8. `risk_flags[]` from safety filter pipeline
9. `confidence_score` from transcript + audio + signal completeness
10. `reuse_potential` for evergreen scheduling

### Top 10 Mistakes to Avoid
1. Black-box scoring with no explanation — creates zero trust
2. Treating strategy-only clips as failures
3. Optimizing for desktop before mobile
4. Shipping without caption style presets
5. Letting users scroll to find the best clip
6. Rendering clips below readiness threshold
7. Cloning OpusClip without differentiating on depth
8. Single pricing tier for creator + agency
9. Ignoring Whop as a customer + distribution channel
10. Shipping without post order guidance
