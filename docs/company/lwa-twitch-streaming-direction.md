# LWA Twitch and Streaming Direction

## Purpose
This document captures why Twitch and stream workflows are part of the core product direction.

## Direction Law

Twitch is not an optional side note.

Twitch and stream content matter because they open:

- stream highlights
- VOD repurposing
- gaming creator workflows
- live-content energy
- community-driven short-form packaging

## Product Expectations

The product should visibly support:

- `Twitch / Stream` as a source mode
- `twitch` and `stream` as source types
- distinct stream-oriented packaging logic

## Output Expectations

When stream context is supported, LWA should aim to return:

- highlight hooks
- short-form packaging
- post order
- community-aware caption direction
- streamer clip angles

## Boundaries

LWA must not claim:

- private stream bypass
- full live chat ingestion without legal and technical review
- platform automation that does not exist

## Current Repo Alignment

Current repo already references:

- Twitch and stream direction in docs and product language
- some frontend mode and copy foundations

Still needed:

- stronger typed backend source handling
- explicit source protocol mapping
- broader frontend surface visibility beyond isolated copy

## Testing Expectations

- Twitch must be treated as a distinct source path
- blocked platform behavior must fall back cleanly
- stream workflows must not silently degrade into “generic YouTube”
