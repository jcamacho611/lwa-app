# LWA Master Product Direction

## Purpose
This is the master founder-readable product direction for Issue #24.

It exists so the repo, prompts, terminals, docs, backend work, frontend work, sales language, and future guild systems stop drifting.

## Core Product Law

LWA / IWA is not:

- YouTube-first
- Whop-only
- video-only
- a generic clipping dashboard

LWA / IWA is:

> an any-source AI content engine

It accepts, when technically possible:

- video
- audio
- music
- prompts
- Twitch / stream sources
- campaign context
- creator objectives
- public URLs
- uploaded files
- future connector sources

It returns creator-ready packages:

- ranked clips
- hooks
- captions
- thumbnail text
- CTA suggestions
- visuals
- post order
- scores
- score breakdowns
- rendered assets when available
- strategy-only packages when rendering or extraction fails

## Core Phrase

> Any source in. Creator-ready content out.

## Product Experience Truth

The user should feel:

- one source engine
- one command center
- one package-building system
- one AI layer that makes repurposing easier

The user should not feel:

- trapped in a YouTube-only form
- trapped in a Whop-only business model
- trapped in a video-only workflow

## Protocol Layer Rule

Every new feature must map to the protocol layer before it gets built.

That means new work must connect to:

- source protocol
- output package protocol
- event protocol
- connector protocol
- consent protocol
- reward protocol

This keeps LWA from becoming a pile of disconnected integrations.

## Current Priority Order

1. protect working routes and deploy assumptions
2. finish any-source frontend realignment
3. finish any-source backend source handling
4. keep prompt systems aligned
5. preserve rendered-vs-strategy truth
6. keep monetization multi-path, not Whop-only
7. add safe guild / mission foundations only after docs and protocol alignment

## Safety Rules

LWA must never drift into:

- hidden compute
- hidden mining
- hidden AI training
- undisclosed user resource use
- guaranteed earnings
- guaranteed viral or revenue claims
- fake auto-posting
- fake campaign submission
- private-content bypass claims

## Current Repo Alignment

Already real:

- backend clipping and ranking foundation
- rendered-vs-strategy distinction
- score and packaging fields
- prompt and protocol architecture docs
- homepage any-source direction on the web

Still incomplete:

- source handling is not fully normalized for every source type
- Twitch / stream needs stronger typed handling
- audio / music / prompt flows need deeper engine alignment
- the web studio still needs broader source-mode language cleanup

## Testing Expectations

Every major direction change must verify:

- backend tests for source and fallback behavior
- web type-check
- claim safety
- rendered-vs-strategy truth
- optional-field-safe output
