# LWA Design Direction: White, Mac-Style, Character-Led

## Purpose
This document captures the founder-approved visual direction so the frontend stops drifting into generic dark SaaS.

## Design Law

The primary LWA visual direction is:

- white / pearl / Mac-style base
- clean premium layout
- colorful futuristic accents
- character-girl / Muse presence
- God App energy

Dark mode can exist, but it must not dominate the product identity.

## Visual Priorities

### Base

- pearl and white canvas
- clean card surfaces
- premium spacing
- Mac-like clarity

### Accents

- iris
- pink
- blue
- emerald
- amber

### Character Presence

The product should feel character-led, not logo-only.

Expected direction:

- character girl or Muse hero
- assistant or strategist presence
- subtle world energy
- premium, original, non-IP-copy visual identity

### Avoid

- generic dark crypto dashboard energy
- cluttered SaaS card piles
- overbearing Whop-style purchase shell
- overdesigned fantasy darkness overwhelming utility

## Product Surface Expectations

### Homepage

Should show:

- any-source product truth
- source modes
- character or Muse presence
- multi-path CTA system

### Studio

Should show:

- source command
- ranked results
- rendered proof
- strategy-only fallback
- packaging clarity

## Current Repo Alignment

Current repo has:

- pearl / blush / iris tokens
- some character and god-style assets
- a homepage moving closer to the any-source pearl direction

Still missing:

- one clearly dedicated character-girl hero asset
- consistent light-first application across the whole web experience
- a complete Muse-01 component language

## Safety Rules

- no copied anime IP
- no fake features as decoration
- no dark-only identity by accident
