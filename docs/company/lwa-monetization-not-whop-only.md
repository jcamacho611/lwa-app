# LWA Monetization Direction: Not Whop-Only

## Purpose
This document prevents the company from drifting into “Whop is the business.”

Whop is one monetization option.
It is not the whole company.

## Monetization Law

LWA must support multi-path monetization over time.

## Allowed Monetization Directions

- Whop access
- direct support
- custom clip packs
- demo calls
- referral or partner programs
- Twitch / stream creator paths
- sponsor paths
- investor conversations
- future subscriptions
- future community or guild perks

## Frontend Implications

The UI should support language like:

- Generate from source
- Request custom clip pack
- Support the build
- Book demo
- Join creator / referral program
- Upgrade on Whop

Whop should stay visible, but it must not dominate the company story.

## Sales Implications

Operators should not pitch LWA as:

- a Whop-only tool
- a campaign-payout machine
- a guaranteed monetization shortcut

Operators should pitch it as:

- an any-source content engine
- a creator workflow accelerator
- a packaging and repurposing engine

## Current Repo Alignment

Current repo already has:

- multi-path money link helpers
- Whop as one upgrade path
- broader money CTA direction on the homepage

Still needed:

- broader business copy updates across other surfaces
- backend and docs language that does not center Whop by default

## Safety Rules

Forbidden claims:

- guaranteed earnings
- guaranteed payouts
- passive income
- campaign submission automation
- fake payout tracking
