Define the founding team Justin needs to build LWA Worlds.

1. clipping hardening
2. Director Brain
3. frontend command center

5. marketplace safety
6. Realms retention
7. optional proof/cosmetic blockchain

## Role scorecards

### Architect of Realms

Owns:

- product system map
- build sequencing
- feature boundaries
- marketplace/Realms/frontend alignment

First 30 days:

- lock MVP scope
- convert roadmap into engineering tickets
- prevent fantasy systems from breaking clipping launch

Red flags:

- wants to restart the repo
- wants to build everything at once
- ignores live product constraints

### Hand of the Director

Owns:

- FastAPI/backend routes
- Next.js/web app
- SQLite/current store and future Postgres transition
- Railway deploy stability
- tests and CI discipline

First 30 days:

- harden generation
- merge Director Brain support safely
- stabilize Whop entitlement verification
- protect `/v1/generate`, `/v1/jobs`, uploads, and health routes

Interview questions:

1. How do you add a new service layer without breaking an existing route?
2. How do you make webhook processing idempotent?
3. How do you test fallback behavior when an external provider fails?

### Forgemaster of Signals

Owns:

- source ingest
- metadata extraction
- transcript segmentation
- Director Brain scoring
- hook/caption engines
- quality gate
- render recovery

First 30 days:

- implement algorithm modules
- add tests for scoring/fallbacks
- make rendered vs strategy-only output honest

Interview questions:

1. How would you clip a 60-minute podcast into 8 shorts?
2. What happens when transcript extraction fails?
3. How do you prevent a render failure from becoming a product failure?

### Veilwright

Owns:

- premium web shell
- result hierarchy
- rendered/raw/strategy-only lanes
- mobile-safe layout
- creator-native language

First 30 days:

- clean generate flow
- make best clip obvious
- make limitations honest
- make CTA path clear

Red flags:

- generic SaaS templates
- overexplained UI
- fake editor controls

### Auditor of the Glass Synod

Owns:

- Whop/Stripe operations
- refund and dispute policy
- payout readiness
- fraud prevention
- claim safety

First 30 days:

- verify Whop flow
- write refund/dispute notes
- define what cannot be sold yet

### Loremaster of the Realms

Owns:

- future retention system
- classes/factions/quests
- no pay-to-win
- XP earned from real usage only

First safe task:

- static Realms content plan only, no blockchain, no XP economy tied to money

### Sigilbearer

Owns:

- optional proof-of-creation
- off-chain proof records first
- Merkle dry run later
- cosmetic-only relic policy

Hard rules:

- no NFT feature unlocks
- no token sale
- no fractionalization
- no ROI/yield language

### Keeper of the Charter

Owns:

- privacy
- terms
- FTC disclosures
- refund policy
- marketplace safety
- AI/content risk

Hard rules:

- no guaranteed virality
- no guaranteed income
- no investment claims
- no unverifiable payment claims
