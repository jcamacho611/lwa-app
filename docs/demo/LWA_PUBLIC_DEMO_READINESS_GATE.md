# LWA Public Demo Readiness Gate

## Purpose

This gate decides whether the public demo is honest enough to show as the first-session LWA experience.

## Checks

| Area | Status | Notes |
|---|---|---|
| First impression | PASS | The route explains the loop quickly. |
| Lee-Wuh clarity | PASS | Lee-Wuh is the guide, not decoration. |
| Source flow | PASS | Source -> clips -> proof remains visible. |
| Rendered vs strategy-only | PASS | Recovery is shown as a real path. |
| Proof CTA | PASS | Proof and export are visible next steps. |
| Signal Sprint | PASS | The game layer is small and understandable. |
| Marketplace | BLOCKER | Keep it teaser-only until the real path is wired. |
| Payouts | BLOCKER | No real payouts or crypto claims in the demo. |
| Social posting | BLOCKER | No direct posting automation claims. |
| Mobile sanity | PASS | The demo must stay readable on smaller screens. |
| Performance | PASS | No large blocking assets are required. |

## Allowed demo language

- upload-first workflow
- ranked clip packages
- rendered vs strategy-only separation
- proof and recovery
- mission progression
- public demo loop

## Disallowed demo language

- guaranteed revenue
- guaranteed viral results
- direct posting automation
- payout automation
- crypto rewards
- hidden mining

## Ship rule

Do not label the public demo production-ready until the route, build, and validation gates pass together.
